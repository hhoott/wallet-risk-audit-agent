Know the risk of a counterparty address before you transact.

CROO Address Intel Agent checks Ethereum and compatible-chain addresses, identifies official contracts when evidence supports it, grades risk, and returns a shareable report.

What the agent solves

The user supplies a transaction counterparty or any address. The agent answers: should I trust it?

1

Collect evidence

Read-only RPC and Etherscan calls inspect address type, contract metadata, transactions, balances, approvals, and warnings.

2

Ask the LLM

The prompt receives the evidence log, not a hardcoded address list, and returns badge, verdict, risk, reasons, and cited fields.

3

Deliver result

The verdict is written back into JSON, A2A delivery, API response, and the web report page.

LLM is not a decorative summary

The model output becomes the final badge and risk classification.

Evidence sent to LLM

contractMeta: { name: "SwapRouter", verified: true }
walletActivity: { txCount: 10000 }
approvals: []
scannerWarnings: []

The prompt asks for verdict, risk level, official flag, blacklist flag, reasons, authorization risks, and evidence used.

Guardrail for official labels

EOA official=true is rejected unless the evidence log has an explicit source or explorer label.

This prevents the model from marking a famous wallet as official just from memory or public reputation.

The LLM may identify low risk, but the official badge is withheld for EOAs unless the evidence log contains an official-source signal.
Normal wallet signals
The report shows EOA type, observed transaction history, no contract-code risk, and no explicit blacklist or scanner warning in the saved evidence.

The agent separates "low apparent risk" from "official", which is important for preventing overconfident UI badges.
