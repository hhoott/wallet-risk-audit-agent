How it works.

  1. 1

    You place an order

    Enter a wallet and pick a tier. The portal negotiates a CAP order with the audit agent.

  2. 2

    Payment is attempted

    If USDC escrow succeeds, CAP settlement is recorded; in demo mode payment failure falls back locally.

  3. 3

    The agent audits, read-only

    It reads public Ethereum data — approvals, contracts, assets, transactions — and never touches keys.

  4. 4

    You get a report

    A health score, risk findings, and revocation links you confirm in your own wallet.

Audited chain: Ethereum Mainnet (read-only) · Settlement: USDC on Base via CAP.

This is a read-only analysis service. It never accesses your private keys or seed phrase and never initiates a transaction. Revocation is offered only as a link for you to confirm in your own wallet.
